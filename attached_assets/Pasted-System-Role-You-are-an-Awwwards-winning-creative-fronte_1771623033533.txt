System Role: You are an Awwwards-winning creative frontend developer specializing in highly interactive, physics-driven web experiences. Your code is clean, performant, and heavily utilizes micro-interactions to tell a story.

Task: Build a React component (using Tailwind CSS and Framer Motion) for a side-by-side comparison section titled "Your DMs Are Not a Sales System." This section must contrast a chaotic, anxiety-inducing "Old Way" (Left Card) with a smooth, automated, flow-state "Zatch Way" (Right Card) through interactive mouse tracking.

Visual Architecture:

Background: Deep immersive black (#050505) with a very faint, dark gray engineering grid (bg-[linear-gradient(to_right,#111_1px,transparent_1px),linear-gradient(to_bottom,#111_1px,transparent_1px)] bg-[size:2rem_2rem]).

Headline: "Your DMs Are Not a Sales System." Text is white, but add a vibrant, glowing neon green text-shadow/outline effect to it. Subtext below: "Stop managing conversations. Start closing deals."

Layout: Two large, glassmorphism cards side-by-side (flex-col on mobile, flex-row on desktop).

Interaction Logic 1: The Left Card ("The Old Way" - Red / Anxiety)

Aesthetic: Dark muted red border. Red glowing accents.

The Content: Title "The Old Way (Chaos & Manual Work)". Bullet points: "Closing one sale takes 15+ mins", "Endless DM back & forth", "Chasing payments manually", "Ghosted by buyers constantly".

The Anxiety Mechanics (Crucial):

Implement mouse tracking inside this specific card. Calculate the velocity/speed of the user's mouse.

If the user moves their mouse quickly inside the red card, trigger a function that spawns "Anxiety Bubbles" (absolute positioned divs) at random coordinates near the cursor.

These bubbles should be small, dark red, glowing pills containing frantic text like: "Price??", "Available?", "Last price??", "Pls reply fast".

Animation: The bubbles should scale up from 0, float upward slightly, and dissolve (opacity 0) within 1.5 seconds using Framer Motion <AnimatePresence>. Limit the max bubbles on screen to 15 to prevent DOM lag.

Glitch Effect: When the user hovers over the bullet point "Endless DM back & forth", apply a CSS chromatic aberration glitch effect to the text (rapidly shifting red and cyan text-shadows).

Interaction Logic 2: The Right Card ("The Zatch Way" - Green / Flow State)

Aesthetic: High-contrast neon green border (#39FF14). Inside is a sleek, dark green glassmorphism background.

The Content: Title "The Zatch Way (Automated & Instant)". Bullet points: "One live stream = Hundreds of potential sales", "Negotiate instantly with auto-counter offers", "Payments collected inside the app", "Zero manual tracking needed".

The Flow State Mechanics (Crucial):

When the cursor enters this card, everything feels smooth and controlled. No chaotic popups.

Hover Effect 1: When hovering over the bullet "One live stream = Hundreds of potential sales", a sleek frosted-glass widget smoothly slides out next to it. Inside the widget, a number rapidly counts up from 124 to 890 (using Framer Motion's useSpring or a simple setInterval), representing live viewers.

Hover Effect 2: When hovering over "Payments collected inside the app", the green checkmark icon next to it does a highly satisfying, bouncy scale-up animation, and a small green tooltip drops down saying "+ ₹4,500 Deal Closed".

Background effect: Use Framer Motion to create a subtle, slowly sweeping green radial gradient that follows the user's cursor only while inside this card, making it feel like a flashlight illuminating a server board.

Technical Requirements:

Use framer-motion for all animations, element mounting/unmounting (the bubbles), and the spring physics.

Use lucide-react for the X icons (red) and Check icons (green).

Ensure the component is fully self-contained in one file for easy testing. Include all necessary state hooks (useState, useEffect, useRef).

Make it fully responsive. On mobile, disable the "Anxiety Bubble" mouse tracking and just rely on the visual hover/tap states.
